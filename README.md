# Hive2Browse

A minimal Qt6-based web browser (HiveBrowser) using Qt WebEngine. This README explains the dependencies and the shortest path to build and run the project, with an example for Void Linux (your environment).

---

## Quick summary

- Required: Qt 6 with the WebEngine module, a C++ toolchain (g++), qmake (qmake6), and make.
- Tested: this repository was generated with qmake and Qt6; the Makefile in-repo was generated by qmake (qmake6).

---

## Void Linux (recommended steps)

1. Update package index:

```sh
sudo xbps-install -S
```

2. Install packages. Package names can vary between Void repos — if one of these is missing, search with `xbps-query -Rs <name>`.

```sh
sudo xbps-install -S gcc g++ make pkgconfig qmake6 qt6 qt6-webengine qt6-tools
```

Note:
- `qmake6` is the qmake binary for Qt6 on many distros; if your system provides `qmake` for Qt6 check `qmake -v`.
- If `qt6-webengine` does not exist under that name, search for packages containing "webengine" (e.g. `xbps-query -Rs webengine`).

## Debian/Ubuntu (example)

```sh
sudo apt update
sudo apt install build-essential qt6-base-dev qt6-webengine-dev qmake qt6-qmake pkg-config
```

On older Debian/Ubuntu you may need `qtwebengine5-dev` and `qmake-qt5` if Qt6 packages aren't available.

## macOS (Homebrew) (example)

Homebrew packages for Qt6 are typically `qt@6`. Qt WebEngine may not be available in all brew bottles; building WebEngine from source is large. If you have Qt6 and WebEngine installed, use the same qmake/make flow.

```sh
brew install qt@6
# make sure qmake from qt@6 is on your PATH (or call it via $(brew --prefix qt@6)/bin/qmake)
```

## Build & run (shortest path)

From the repo root:

```sh
# generate Makefile (a Makefile is already present; run this if you changed the .pro or don't trust the generated Makefile)
qmake6 hive2browse.pro
make
# then run
./hive2browse
```

If `qmake6` is not available but `qmake` points to Qt6, use `qmake` instead.

## Notes, recommendations, and housekeeping

- The repository currently includes built artifacts (`hive2browse`, `*.o`). It's best practice to remove these from the repository and add a `.gitignore` to avoid shipping binaries in source control. I can prepare a commit to remove them and add a `.gitignore` if you want.

- Security: do not run binaries from unknown sources. Build from source on your machine when possible.

- The code expects Qt WebEngine available at runtime (libQt6WebEngineWidgets, libQt6WebEngineCore). If you see runtime errors about missing shared libraries, install the matching Qt6 runtime packages for your distro or set `LD_LIBRARY_PATH` to point to your Qt lib directory.

---

## Troubleshooting

- If the build fails with undefined references to WebEngine symbols, verify `qt6-webengine` (or your distro's equivalent) is installed and that `qmake6` picks up Qt6 (run `qmake6 -v`).
- If qmake generates a different Makefile, run `make clean` then re-run `qmake6`.
- On Wayland/Wayland-only systems you may need additional Qt platform packages (e.g. `qt6-wayland` or xcb/x11 compatibility depending on your environment).
